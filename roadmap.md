# Roadmap


## План рефакторинга архитектуры плеера

### 1. Переход на Zustand [ВЫПОЛНЕНО]
- [x] Замена `PlayerContext` на `zustand`.
- [x] Создание независимого хранилища и хука `usePlayerStore`.

### 2. Реорганизация файлов и компонентов [ВЫПОЛНЕНО]
- [x] Миграция кода в `src/`.
- [x] Декомпозиция монолита плеера на автономные папки (`PlayerControls`, `MobilePlayer`, `LyricsDrawer`).
- [x] Внедрение персональных CSS модулей для каждого компонента.

### 3. Независимая загрузка метаданных и кеширование [ВЫПОЛНЕНО]
- [x] Выделение извлечения цветов в хук `useTrackColor` с кешем.
- [x] Обеспечение независимости Lyrics от загрузки аудиоконтекста.
- [x] Исправление логики скролла в текстах.

### 4. Улучшение работы Media Session API [ВЫПОЛНЕНО]
- [x] Устранение конфликтов Web Audio API с системным уведомлением.
- [x] Отказ от использования `dummyAudio` и `setPositionState` (решение проблемы "исчезающей паузы").
- [x] Обеспечение стабильной работы сессии на мобильных устройствах без ущерба для визуализации.

### 5. Регулятор громкости [ВЫПОЛНЕНО]
- [x] Внедрение настройки уровня громкости в стор и UI.
- [x] Синхронизация громкости с `ReactHowler`.

### 6. Оптимизация производительности [ВЫПОЛНЕНО]
- [x] Переход на атомарные селекторы Zustand.
- [x] Изоляция компонентов с высокой частотой обновлений (прогресс-бар, таймеры).

### 7. Глобальная стабилизация движка воспроизведения [ВЫПОЛНЕНО]
- [x] Анализ и устранение утечки памяти HTML5 Audio (ошибка `pool exhausted`).
- [x] Устранение бесконечного цикла обновлений React (`Maximum update depth exceeded`) через удаление нестабильных механизмов Recovery.
- [x] Решение проблемы параллельных потоков и конфликтов интеграции с нативной Media Session ОС.
- [x] Стабилизация работы `AudioMotionVisualizer` в условиях React Strict Mode без сброса контекстов.
